⚡ 4 QUICK START BUTTONS
───────────────────────────────────────────────────────────────────────────────

1. ▶  RUN FULL TEST
   ├─ Time: 15-30 min
   ├─ Does: Load data → Validate 111 features → Analyze 9 triggers
   │         Train XGBoost → Test 9 scenarios → Generate report
   ├─ Output: logs/ml_test_report_YYYYMMDD_HHMMSS.json
   └─ Use when: Want comprehensive ML diagnostics

2. 📊  GENERATE TEST DATA  
   ├─ Time: 2-5 min
   ├─ Does: Create synthetic 3-min candles for 6 market scenarios
   │         (Trending Bull/Bear, Consolidation, Reversal, Gap, Volatile)
   ├─ Output: test_data/ with 6 CSV files
   └─ Use when: Need realistic data without waiting for market

3. 📤  EXPORT OPTIONS DATA
   ├─ Time: 1-3 min
   ├─ Does: Export daily options prices + snapshots + ML features
   │         (11K EOD rows + 1.5K snapshots + pre-computed features)
   ├─ Output: logs/options_data_exports/ with 4 files
   └─ Use when: Need latest options data for training

4. ✓  VALIDATE FEATURES
   ├─ Time: 2-5 min
   ├─ Does: Check all 111 features in real training data
   │         Detect NaN/missing values, analyze ranges
   ├─ Output: Feature quality report
   └─ Use when: Verify data quality before training

🔄 HOW THE CONNECTION WORKS
───────────────────────────────────────────────────────────────────────────────

                    MAIN UI THREAD
                         │
                         │
        ┌────────────────▼────────────────┐
        │   User Clicks Button            │
        │   (ML TESTER Tab)              │
        └────────────────┬────────────────┘
                         │
                    ✅ BUTTON ENABLED
                by: _disable_buttons()
                         │
         ┌───────────────▼───────────────┐
         │ Create MLTestWorker Thread    │
         │ (Background Thread)           │
         └───────────────┬───────────────┘
                         │
        ┌────────────────▼────────────────┐
        │ Worker.run_full_test()          │
        │ - Loads data                   │
        │ - Validates features           │
        │ - Analyzes triggers            │
        │ - Trains model                 │
        │ - Tests scenarios              │
        └────────────────┬────────────────┘
                         │
        ┌────────────────▼────────────────┐
        │ Emit Signals to UI Thread       │
        │ - progress.emit(msg)           │
        │ - test_complete.emit(result)   │
        │ - error.emit(error_msg)        │
        └────────────────┬────────────────┘
                         │
        ┌────────────────▼────────────────┐
        │ UI Slots Handle Signals         │
        │ - _on_progress() → Log update  │
        │ - _on_test_complete() → Table  │
        │ - _on_error() → Error log      │
        └────────────────┬────────────────┘
                         │
                    ✅ BUTTON ENABLED
                by: _enable_buttons()
                         │
              Display Results in UI ✅

🧪 WHAT EACH TEST DOES IN DETAIL
───────────────────────────────────────────────────────────────────────────────

1. FULL DIAGNOSTIC
   ├─ Phase 1: Load Data
   │  ├─ Fetch real training data from DB (30 days)
   │  └─ Generate synthetic data (6 scenarios × 60 days each)
   │
   ├─ Phase 2: Feature Validation
   │  ├─ Check all 111 features present
   │  ├─ Calculate NaN% per feature
   │  ├─ Report coverage percentage
   │  └─ Flag missing critical features
   │
   ├─ Phase 3: Trigger Analysis
   │  ├─ For each of 9 triggers:
   │  │  ├─ Count fire frequency
   │  │  ├─ Calculate win rate %
   │  │  └─ Rank by effectiveness
   │  └─ Output: Which triggers actually work
   │
   ├─ Phase 4: Model Training
   │  ├─ Split: 80% train / 20% test
   │  ├─ Train: XGBoost with 111 features
   │  ├─ Metrics: Accuracy, F1, AUC, Precision, Recall
   │  └─ Extract: Top 15 features by importance
   │
   ├─ Phase 5: Scenario Testing
   │  ├─ Test 9 market scenarios:
   │  │  ├─ Bullish/bearish trends
   │  │  ├─ High/low volatility
   │  │  ├─ Expiry day
   │  │  ├─ Early/closing session
   │  │  ├─ With compression/DI
   │  └─ Output: Win rate per scenario
   │
   └─ Phase 6: Report Generation
      └─ JSON file with all results
